# رَفّ — نظام فهرسة وإدارة المكتبات

تطبيق مكتبي احترافي (Desktop App) لفهرسة الكتب وإدارتها، مبني بتقنية Electron، بواجهة عربية بالكامل (RTL)، ويعمل دون الحاجة إلى إنترنت بعد التثبيت. جميع البيانات تُخزَّن محلياً على جهاز المستخدم.

تطوير: **Hzifa33** — https://Hzifa33.github.io

---

## 1) محتويات المشروع

```
raff-app/
├── main.js                 # العملية الرئيسية لـ Electron (نافذة التطبيق + قاعدة البيانات)
├── preload.js               # جسر آمن بين الواجهة وNode.js
├── package.json             # إعدادات المشروع وإعدادات البناء (electron-builder)
├── assets/                  # الشعار، الخلفية، أيقونات التطبيق
├── src/
│   ├── index.html            # هيكل الواجهة
│   ├── css/                  # التصميم (tokens.css / app.css / fonts.css)
│   ├── fonts/                # خطوط Amiri وCairo (محلية، بدون إنترنت)
│   └── js/
│       ├── store.js           # قاعدة بيانات JSON محلية (إضافة/بحث/تعديل/حذف/نسخ احتياطي)
│       ├── icons.js            # مكتبة أيقونات SVG (بدون إيموجي)
│       ├── store-bridge.js     # طبقة تخزين مؤقت في الواجهة
│       ├── ui.js                # نوافذ التأكيد والإشعارات (Toast)
│       ├── views.js             # شاشات التطبيق (لوحة المعلومات، البحث، الإضافة، الإحصائيات، الإعدادات)
│       └── app.js               # التوجيه الرئيسي (Router) وربط الأحداث
└── .github/workflows/build-windows.yml   # بناء تلقائي لملف exe عبر GitHub Actions
```

## 2) المميزات المُنفّذة

- **فهرسة كاملة**: اسم الكتاب، المؤلف، دار النشر، الرقم المرجعي، التصنيف، الطبعة، سنة النشر، عدد النسخ، الحالة (متاح/معار)، اسم المستعير، ملاحظات.
- **بحث متقدم**: بحث فوري أثناء الكتابة، فلترة حسب الحقل (الكل/العنوان/المؤلف/دار النشر/الرقم المرجعي/التصنيف)، فلترة حسب الحالة، وترتيب النتائج.
- **بحث سريع** من الشريط العلوي في أي شاشة (اختصار `Ctrl+K`).
- **إضافة كتاب** بنموذج تحقق من الحقول الإلزامية، مع اقتراحات تلقائية للمؤلفين ودور النشر والتصنيفات المستخدمة سابقاً.
- **لوحة معلومات وإحصائيات**: إجمالي الكتب، المؤلفين، دور النشر، الكتب المعارة، وتوزيع التصنيفات كرسم بياني بسيط بدون مكتبات خارجية.
- **نسخ احتياطي واستيراد**: تصدير JSON كامل، تصدير CSV (متوافق مع Excel)، استيراد نسخة سابقة مع تجاهل الأرقام المرجعية المكررة تلقائياً.
- **حذف جميع البيانات** مع نافذة تأكيد صريحة (منطقة الخطر).
- تصميم مستوحى من الصورتين المرفقتين: لون الخشب الدافئ للرفوف كخلفية للشريط الجانبي، ولون النحاس/الذهبي كلون مميز، متناسقاً مع شعار "رَفّ" الفضي.
- رابط "تطوير Hzifa33" أسفل الشريط الجانبي يفتح `https://Hzifa33.github.io` في المتصفح مباشرة.
- تخزين البيانات في مجلد بيانات المستخدم الخاص بالنظام (لا يُحذف عند إعادة تثبيت التطبيق).

## 3) التشغيل في وضع التطوير

**المتطلبات:** Node.js إصدار 18 أو أحدث (يُفضّل 20).

```bash
cd raff-app
npm install
npm start
```

سيفتح التطبيق كنافذة مكتبية مباشرة على جهازك لتجربته قبل البناء النهائي.

---

## 4) بناء ملف Windows Installer (.exe) — خطوة بخطوة

### الطريقة الأولى: البناء مباشرة من ويندوز (الأسهل والموصى بها)

1. انسخ مجلد المشروع كاملاً إلى جهاز يعمل بنظام Windows.
2. ثبّت [Node.js](https://nodejs.org) (النسخة LTS).
3. افتح موجه الأوامر (Command Prompt أو PowerShell) داخل مجلد المشروع.
4. نفّذ:
   ```bash
   npm install
   npm run dist:win
   ```
5. بعد انتهاء البناء (قد يستغرق دقائق)، ستجد ملف التثبيت هنا:
   ```
   release\رَفّ Setup 1.0.0.exe
   ```
6. شغّل هذا الملف على أي جهاز ويندوز لتثبيت التطبيق (سيظهر اختصار على سطح المكتب وقائمة ابدأ باسم "رَفّ").

> **لماذا يُفضَّل البناء من ويندوز؟** لأن حزمة NSIS (منشئ ملفات exe) مصمّمة خصيصاً لويندوز، والبناء عليه مباشرة يعطي أفضل نتيجة متوافقة دون الحاجة لأي أدوات محاكاة إضافية.

### الطريقة الثانية: بناء تلقائي عبر GitHub Actions (بدون جهاز ويندوز)

تم تجهيز ملف الأتمتة `.github/workflows/build-windows.yml` مسبقاً داخل المشروع. الخطوات:

1. أنشئ مستودع (Repository) جديد على GitHub وارفع إليه مجلد المشروع كاملاً (باستثناء `node_modules`).
2. من تبويب **Actions** في المستودع، اختر workflow باسم **Build Windows Installer**، ثم اضغط **Run workflow**.
3. بعد انتهاء التنفيذ (تلقائياً على خوادم ويندوز من جوجل)، ستجد ملف الـ exe جاهزاً للتحميل من قسم **Artifacts** في نفس صفحة التنفيذ.

هذه الطريقة تعطي نفس نتيجة البناء من ويندوز تماماً، لكنها لا تتطلب أن تملك جهاز ويندوز بنفسك.

### الطريقة الثالثة: البناء من macOS أو Linux باستخدام Wine

إن أردت البناء من جهاز Mac أو Linux مباشرة:

```bash
# تثبيت wine أولاً (مطلوب لبناء exe من غير ويندوز)
# على Ubuntu/Debian:
sudo apt install wine

npm install
npm run dist:win
```

## 5) بناء نسخة Linux (اختياري)

إن أردت أيضاً نسخة تجريبية تعمل على Linux مباشرة (AppImage):

```bash
npm run dist:linux
```

الناتج: `release/رَفّ-1.0.0.AppImage` — ملف تنفيذي واحد لا يحتاج تثبيتاً، فقط امنحه صلاحية التشغيل:

```bash
chmod +x "release/رَفّ-1.0.0.AppImage"
"./release/رَفّ-1.0.0.AppImage"
```

*(تم بناء هذه النسخة واختبارها بنجاح أثناء تجهيز المشروع، للتأكد أن إعدادات `electron-builder` سليمة قبل تسليمه لك.)*

## 6) أين تُحفظ بيانات المكتبة؟

يستخدم التطبيق ملف JSON محلي (بدون قواعد بيانات معقدة تحتاج تجميعاً خاصاً لكل نظام تشغيل)، مما يضمن أن التطبيق يعمل فوراً دون أي أخطاء تثبيت متعلقة بمكتبات أصلية (native modules). مسار الملف:

- **ويندوز:** `%APPDATA%\raff-library\raff-library.json`
- **macOS:** `~/Library/Application Support/raff-library/raff-library.json`
- **Linux:** `~/.config/raff-library/raff-library.json`

يمكن رؤية هذا المسار مباشرة من داخل التطبيق: **النسخ الاحتياطي والإعدادات → معلومات النظام**.

---

## 7) تخصيص الألوان أو الشعار لاحقاً

- الألوان: عدّل المتغيرات في `src/css/tokens.css` (كل الألوان معرّفة كمتغيرات CSS في مكان واحد).
- الشعار/الخلفية: استبدل `assets/logo.png` و`assets/background.jpg` بنفس الاسم للحفاظ على الروابط، ثم أعد تشغيل `npm run dist:win`.
- أيقونة التطبيق (exe/AppImage): استبدل `assets/icon.ico` و`assets/icon.png`.

## 8) استكشاف الأخطاء

| المشكلة | الحل |
|---|---|
| `npm install` يفشل | تأكد من اتصال الإنترنت وأن Node.js مثبّت (`node -v`) |
| نافذة بيضاء عند التشغيل | افتح أدوات المطوّر بـ `Ctrl+Shift+I` وراجع الأخطاء في تبويب Console |
| `npm run dist:win` يفشل بسبب Wine | استخدم الطريقة الثانية (GitHub Actions) بدلاً من ذلك |
| فقدان البيانات بعد التحديث | البيانات محفوظة خارج مجلد التطبيق تماماً (انظر القسم 6)، لذا لا تُحذف عند تحديث أو إعادة تثبيت التطبيق |
